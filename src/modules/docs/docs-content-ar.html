هندسة النظام

النظام مبني كـ pnpm monorepo. جميع واجهات المستخدم مبنية كأصول ثابتة ويتم تقديمها بواسطة خادم Express مركزي على المنفذ 8080.

monorepoworkspace/
├── src/
│   ├── modules/
│   │   ├── dashboard/       # لوحة القيادة المركزية
│   │   ├── skill-matrix/    # تطبيق مصفوفة المهارات
│   │   ├── job-evaluation/  # تقييم الوظائف
│   │   ├── pitch-deck/      # العرض التقديمي
│   │   └── docs/            # هذا التوثيق
│   │
│   └── shared/              # المكونات والخدمات المشتركة
│       ├── components/      # UI Library
│       ├── contexts/        # Lang, Auth contexts
│       └── i18n/            # ملفات الترجمة (EN/AR)

التدفق التشغيلي

دورة "الذكاء الصناعي" من البداية حتى توصيات التدريب المؤتمتة. يضمن هذا التدفق أن كل تقييم يؤدي إلى خطط نمو قابلة للتنفيذ.

1
بدء الحملة
يقوم المسؤولون بتعريف نافذة التقييم، الأقسام المستهدفة، والمهارات النشطة.
2
التقييم النشط
يقوم رؤساء الأقسام والمدراء بإدخال الدرجات (0-4) لأعضاء فريقهم باستخدام الشبكة الصناعية.
3
رؤى الذكاء الاصطناعي والتصنيف
يقوم النظام تلقائياً بحساب المتوسطات المرجحة وتعيين فئات الأداء (A أو B أو C).
4
توليد التدريب
تؤدي الفجوات المحددة إلى إطلاق توصيات تدريب مؤتمتة (فورية، قصيرة المدى، أو طويلة المدى).

قاعدة البيانات — الموظفين (employees)

سجلات الموظفين الرئيسية. ينتمي كل موظف إلى قسم واحد وله فئة أداء (Class) يتم تحديثها بعد كل حملة تقييم.

منطق تسجيل الدرجات

يتم حساب النتيجة النهائية للموظف بناءً على الدرجات الممنوحة لكل مهارة مضروبة في وزن تلك المهارة.

المعادلة: (مجموع (الدرجة × الوزن)) / (مجموع (4 × الوزن)) × 100

مستويات الدرجات

  • 0: لا يمكنه أداء المهمة
  • 1: يتطلب إشرافاً مستمراً
  • 2: يتطلب إشرافاً أحياناً
  • 3: يعمل بشكل مستقل
  • 4: خبير / يمكنه تدريب الآخرين

تصنيف الفئات (Classes)

  • Class A: النتيجة ≥ 85%
  • Class B: النتيجة بين 60% و 84%
  • Class C: النتيجة أقل من 60%
